# 90-Day Agency Growth Flywheel

Last updated: 2026-06-14 02:50 CDT

## Purpose

Define how Kelvar LLC turns the current agency operating system into a repeatable growth loop over the next 90 days.

This is an internal execution map. It does not send outreach, create mailbox drafts, quote prices, select paid vendors, change deployment settings, launch the website, publish proof, or approve auto-send behavior.

## Operating Boundary

- Repository: `KelvarLLC/Business1`
- Workspace: `agency-ops/`
- Primary offer: Lead Intake Command System / Follow-Up Engine
- Current market wedge: service businesses with leaky lead intake, slow first response, inconsistent follow-up, or unclear owner visibility
- Current launch truth: website not launched, outreach not sent, Stage 1 intake not live, proof is demo-only

## The Flywheel

The agency grows by cycling through seven repeatable loops:

1. Identify a painful workflow niche.
2. Package a narrow automation offer.
3. Create a controlled prospect batch.
4. Run draft-first outreach and discovery.
5. Convert qualified interest into scoped delivery.
6. Turn delivery into safe proof.
7. Feed lessons back into the offer, website, outreach, and delivery assets.

Each loop should produce a repository artifact, a clearer decision, or a safer next action.

## 90-Day Execution Arc

### Days 1-15: Stabilize The Operating Base

Primary objective:

Make the agency ready to take controlled external action.

Work to complete:

- Keep the command brief current.
- Keep approval gates visible.
- Keep website launch status honest.
- Prepare first outreach batch for review.
- Prepare Stage 0 and Stage 1 intake readiness.
- Keep the delivery and closeout packages ready.

Current blockers:

- Outreach still needs action-level approval before drafts or sends.
- Website build/preview evidence is not available in this environment.
- Stage 1 intake cannot be called live until handler decision, setup, and QA are complete.

### Days 16-30: Run The First Controlled Revenue Test

Primary objective:

Start the first small outbound test once approved.

Actions after approval:

- Create first-batch mailbox drafts for review.
- Revise drafts based on user review.
- Send only after separate explicit send approval.
- Track reply status in the six-lead tracker.
- Classify replies using the reply response kit.
- Use discovery script for interested leads.
- Update proposal-readiness snapshot.

Learning targets:

- Which message angle earns response.
- Which objections appear first.
- Which lead types feel closest to pain.
- Whether the offer language needs tightening.

### Days 31-45: Convert Interest Into Scoped Work

Primary objective:

Move one qualified conversation toward a proposal or paid pilot.

Actions:

- Complete discovery notes.
- Score proposal readiness.
- Create proposal outline when facts support it.
- Keep pricing gated until approved.
- Prepare delivery plan from the confirmed scope.
- Avoid overbuilding the first engagement.

Success target:

At least one prospect reaches discovery-needed, proposal-outline-ready, or proposal-draft-ready with enough facts to avoid guessing.

### Days 46-60: Deliver The First Narrow Win

Primary objective:

Complete one bounded phase-one workflow if a project is won.

Actions:

- Confirm one source, one tracker, one owner group, one status model.
- Build or coordinate implementation inside approved tools.
- Test happy path and edge cases.
- Complete handoff packet.
- Record limitations and phase-two opportunities.

Success target:

The client can operate the workflow and sees clearer lead ownership, next action, and follow-up timing.

### Days 61-75: Capture Proof And Improve The Website

Primary objective:

Use approved delivery evidence to strengthen trust.

Actions:

- Run 14-30 day follow-up review.
- Ask for proof permission only if client satisfaction is clear.
- Keep proof anonymized unless named permission is explicit.
- Update website and proposal proof only after approval.
- Keep demo proof separate from client results.

Success target:

At least one approved process-proof or result-proof asset exists, or a clear decision says no public proof yet.

### Days 76-90: Expand The Repeatable System

Primary objective:

Use the first cycle to improve scale.

Actions:

- Review outreach learning.
- Refine lead criteria.
- Update offer scope menu.
- Improve website FAQ and objection handling.
- Decide whether to expand in Dallas home services or test a second niche.
- Create the next controlled prospect batch.

Success target:

The agency has a clearer niche, a sharper offer, a proven delivery packet, and a repeatable outreach-to-discovery-to-delivery loop.

## Weekly Operating Rhythm

Monday:

- Read current command brief.
- Review blockers and gated decisions.
- Select one revenue action and one safe operations action.

Tuesday-Wednesday:

- Execute approved outreach, discovery, proposal, delivery, or website-readiness work.
- Update repo artifacts as work progresses.

Thursday:

- Review learning log, lead tracker, proposal readiness, and website blockers.
- Adjust messaging or delivery assets if patterns emerge.

Friday:

- Create or update weekly operator review if meaningful activity occurred.
- Update memory, command brief, and next move board if priorities changed.

## Decision Gates

Continue without approval only when:

- Work stays inside `agency-ops/`.
- It is non-destructive.
- It improves readiness, documentation, proposal quality, delivery quality, QA, proof capture, or operating continuity.

Require explicit action-level approval for:

- Creating mailbox drafts.
- Sending outreach or follow-ups.
- Quoting final pricing.
- Sending proposals.
- Selecting paid tools or vendors.
- Changing deployment settings or launch status.
- Publishing client proof.
- Enabling auto-send.
- Touching any repository outside `KelvarLLC/Business1`.

## Grand Operating Principle

Ambition becomes useful when it compounds into a system.

The agency should not merely create documents. It should create a machine that notices opportunities, packages them, pursues them carefully, delivers cleanly, captures proof honestly, and becomes sharper after every cycle.

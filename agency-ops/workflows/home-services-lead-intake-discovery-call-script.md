# Home Services Lead Intake Discovery Call Script

## Purpose

Give Kelvar LLC a practical discovery-call structure for qualified replies or website inquiries related to the Lead Intake Command System / Follow-Up Engine offer.

This script is designed for Dallas/Fort Worth home-service prospects across:

- HVAC
- Roofing
- Landscaping

It should be used after a prospect shows interest, asks for more information, or agrees that mapping the current quote/estimate intake process would be useful.

## Important Guardrails

Do not use this call to:

- Quote final pricing.
- Promise guaranteed revenue, guaranteed response times, or guaranteed close rates.
- Recommend legal, insurance, compliance, or regulated-data practices.
- Offer auto-send behavior as the default first phase.
- Commit to changing production systems before scope and approval.
- Claim client proof or case-study results that do not exist.

Use this call to:

- Understand the current intake path.
- Identify the simplest workflow gap.
- Decide whether the opportunity is discovery-needed, proposal-outline-ready, proposal-draft-ready, nurture, or no-fit.
- Route the prospect into the right scope-menu path.

## Call Shape

Recommended length: 20 to 30 minutes

Recommended tone: practical, specific, and diagnostic

Call objective:

> Find out whether new quote, estimate, or contact requests are being manually copied, sorted, tracked, or followed up in a way that creates delays, missed handoffs, or unclear ownership.

## Opening

Use this structure:

> Thanks for taking the time. I am not here to push a full CRM rebuild. The goal is to understand how new estimate or quote requests move from your website, phone, or inbox into the next action. If there is a small workflow that can make routing or follow-up clearer, I will point that out. If there is not, I will say that too.

Confirm:

- Time available
- Role of the person on the call
- Whether they handle new requests directly
- Whether the call should focus on one intake path or several

## Section 1: Current Intake Sources

Ask:

1. Where do new estimate, quote, or service requests come from today?
2. Which sources matter most: website form, phone, email, text, ads, referrals, marketplace platforms, or repeat customers?
3. Which one causes the most manual follow-up?
4. Which one is most valuable if handled quickly?

Listen for:

- Form-to-email only
- Phone notes outside the CRM
- Multiple inboxes
- Missed calls or voicemails
- Manual copy/paste into spreadsheets
- Requests split across website, email, and field staff

Record:

- Primary intake source
- Secondary intake sources
- Highest-value source
- Messiest source

## Section 2: First Response And Ownership

Ask:

1. Who sees a new request first?
2. Who is responsible for the first response?
3. Is there a target response window?
4. What happens if the first person is busy, in the field, or away?
5. How does the team know a request has not been contacted yet?

Listen for:

- One person manually checking email
- Owner bottleneck
- Admin bottleneck
- Dispatcher bottleneck
- No visible open-request list
- No backup when the owner/admin is unavailable

Record:

- First owner
- Backup owner
- Response window
- Current tracking method
- Follow-up failure risk

## Section 3: Routing And Classification

Ask general questions:

1. Do different request types need different next steps?
2. What information tells you who should handle a request?
3. Are some requests more urgent or higher value?
4. Are any requests missing details that slow down follow-up?

HVAC-specific prompts:

- Are repair, replacement, maintenance, and estimate requests routed differently?
- Are commercial refrigeration or emergency categories treated differently?
- Does the team need to know system count, location, or service type before follow-up?

Roofing-specific prompts:

- Are residential, commercial, repair, storm-damage, gutter, or insurance-related inquiries handled differently?
- Are photos reviewed before scheduling?
- Does roof age or project type change the follow-up path?

Landscaping-specific prompts:

- Are residential, commercial, tree-care, maintenance, design/build, and irrigation requests routed differently?
- Does budget range change qualification?
- Does property type or location change who follows up?

Record:

- Request categories
- High-priority categories
- Missing fields
- Routing owner or rule
- Classification opportunity

## Section 4: Follow-Up Rhythm

Ask:

1. After the first response, how do you track whether the prospect answered?
2. How do you know if an estimate has been scheduled?
3. How do you know if a quote was sent but not followed up?
4. Are follow-ups calendar-based, memory-based, CRM-based, or spreadsheet-based?
5. Where do prospects most commonly stall?

Listen for:

- Follow-up depends on memory
- No reminders after quote sent
- No status view
- Requests marked complete too early
- Prospects stuck between contacted and scheduled

Record:

- Current statuses
- Follow-up trigger
- Reminder method
- Most common stall point
- Desired visibility

## Section 5: Current Tools

Ask:

1. What tools are involved today?
2. Does the website form go into email, CRM, field-service software, a spreadsheet, or somewhere else?
3. Are phone calls and form requests tracked in the same place?
4. Are there tools you want to keep no matter what?
5. Are there tools the team avoids or finds annoying?

Listen for:

- CRM exists but is inconsistently used
- Form submissions go only to email
- Field-service software handles jobs but not early leads well
- Spreadsheets used as backup
- Manual double entry

Record:

- Website/form tool
- CRM or field-service tool
- Calendar/scheduling tool
- Spreadsheet or manual tracker
- Tool constraint

## Section 6: First Win

Ask:

1. If we only improved one thing in the next 2 to 3 weeks, what would be most useful?
2. Would you rather improve first-response visibility, routing, follow-up reminders, quote status, or reporting?
3. What would make this worth trying without disrupting your current process?
4. Who would need to approve or maintain it?

Possible first-win categories:

- Workflow blueprint only
- Single-source intake tracker
- Request classification and routing
- Follow-up reminders
- Quote or estimate status board
- Weekly open-lead summary
- Internal draft-response support

Record:

- Preferred first win
- Business reason
- Decision owner
- Maintenance owner
- Must-avoid disruption

## Section 7: Scope Menu Routing

Use `agency-ops/offers/lead-intake-scope-menu.md` after the call.

### Blueprint Path

Use when:

- The prospect is curious but not ready to build.
- The process is unclear.
- Multiple tools or stakeholders are involved.
- The best first step is a workflow map and recommendation.

### Single-Source Build Path

Use when:

- One intake source is clearly the biggest pain.
- The workflow can be built around one form, inbox, or tracker.
- The first win is routing, tracking, or reminders.

### Follow-Up Visibility Upgrade Path

Use when:

- They already capture leads but lose visibility after first contact.
- The main gap is follow-up timing, quote status, or owner visibility.

## Close

Use this structure:

> This is helpful. I am going to turn what you shared into a simple recommendation: either a workflow map, a small first build, or a follow-up visibility upgrade. I will keep it bounded around the intake path we discussed, not a full-system rebuild. If there are missing details, I will ask for those before suggesting scope.

Confirm:

- Best contact for follow-up
- Whether they want a written summary
- Whether there is another stakeholder
- Whether there are tools or screenshots they can share later
- Whether sensitive customer data should be avoided in examples

## Proposal Readiness Check

After the call, use `agency-ops/proposals/lead-intake-proposal-decision-scorecard.md`.

Likely outcomes:

- Proposal-draft-ready: enough detail to write a bounded proposal.
- Proposal-outline-ready: direction is clear but a few details are missing.
- Discovery-needed: process remains unclear.
- Nurture or close: no urgent pain, no fit, or no decision path.

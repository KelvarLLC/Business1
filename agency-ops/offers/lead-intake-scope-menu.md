# Lead Intake Scope Menu

## Purpose

Give the Lead Intake Command System / Follow-Up Engine offer a simple scope menu for discovery calls, reply handling, and proposal drafting.

This is not final pricing. It is a packaging guide that keeps the first sale bounded, practical, and easy to explain.

## Packaging Principle

Start with the smallest workflow that makes missed leads harder to ignore.

Do not sell a full CRM rebuild as the first project unless the client explicitly asks for it and the scope is reviewed separately.

## Package 1: Intake Map And Quick-Win Blueprint

### Best Fit

Use when the prospect is curious but not ready to commit to implementation, or when the workflow is still unclear.

### Client Problem

The business suspects leads are being missed or followed up inconsistently, but nobody has mapped the current intake path clearly.

### Included

- Review one primary lead source.
- Map the current intake and follow-up path.
- Identify where manual copying, missed ownership, or stale follow-up happens.
- Define required lead fields and a simple stage model.
- Recommend the smallest useful automation path.
- Produce a short blueprint that can become a proposal or implementation scope.

### Not Included

- Live automation build.
- CRM migration.
- Auto-sending external messages.
- Multi-source routing.

### Deliverable

A concise workflow blueprint with:

- Primary lead source.
- Current process map.
- Missing or duplicated steps.
- Recommended tracker destination.
- Recommended follow-up mechanism.
- Phase-one implementation scope.
- Risks and dependencies.

### Proposal Language

"We will map one lead intake path and define the smallest automation that would make new inquiries visible, assigned, and easier to follow up. This gives you a clear implementation plan without committing to a full system rebuild."

## Package 2: Single-Source Lead Intake System

### Best Fit

Use when the prospect has one high-value intake source and wants a working system.

### Client Problem

New estimate or service requests arrive, but first response, owner assignment, and follow-up tracking depend too much on memory or inbox checking.

### Included

- Connect or simulate one primary lead source.
- Normalize incoming lead details into a tracker or CRM-ready structure.
- Add required field handling.
- Add stage, owner, next action, and follow-up due fields.
- Add internal reminders or draft-response prompts.
- Add a simple owner/admin view.
- Test happy path and key edge cases.
- Provide handoff notes.

### Not Included

- Multiple intake sources.
- Full CRM migration.
- Advanced routing across locations or departments.
- Auto-send behavior unless separately reviewed and approved.
- Paid ads or attribution reporting.

### Deliverable

A working phase-one lead intake workflow with one source, one tracker, one follow-up rhythm, and a handoff note.

### Proposal Language

"Phase one focuses on one intake source, one tracker, and one follow-up rhythm. The goal is to make sure new inquiries are captured, visible, assigned, and followed up consistently before expanding the system."

## Package 3: Follow-Up And Owner Visibility Upgrade

### Best Fit

Use when the client already has a CRM, sheet, or lead tracker but follow-up still goes stale.

### Client Problem

The business has a place where leads live, but the team still lacks clear reminders, stale-lead visibility, or owner-level reporting.

### Included

- Review the existing tracker or CRM process.
- Define stale-lead conditions.
- Add follow-up reminder logic or draft-response prompts.
- Add owner/admin views for new, waiting, qualified, and stale leads.
- Add manual override and pause rules.
- Document how the team should review and update statuses.

### Not Included

- Replacing the existing CRM.
- Rebuilding all intake sources.
- Auto-sending messages without approval.
- Deep reporting or revenue attribution.

### Deliverable

A follow-up control layer that makes overdue lead actions visible and easier to manage.

### Proposal Language

"We will keep your existing lead system and add a practical follow-up layer around it: stale-lead visibility, next-action reminders, and a simple owner view of what needs attention."

## Discovery-To-Scope Rules

Use these rules after discovery:

- If the client cannot name one primary lead source, start with Package 1.
- If the client has one obvious source and no reliable tracker, recommend Package 2.
- If the client already has a tracker but follow-up is inconsistent, recommend Package 3.
- If the client wants multiple sources, recommend one source first and queue the rest as phase two.
- If the client wants auto-send, start with reminders or drafts unless exact message rules and approvals are clear.
- If regulated or sensitive data is involved, keep the scope internal, conservative, and human-reviewed.

## Pricing Conversation Guidance

Do not quote a final price before scope is known.

Safe response when asked about cost:

```text
The price depends mostly on whether we are mapping the process, building one intake path, or adding a follow-up layer around an existing tracker. I would first confirm the lead source, tracker destination, and follow-up mechanism, then scope the smallest useful version.
```

Internal note:

- Add final pricing only after the user sets pricing defaults or approves package pricing.
- Do not publish pricing publicly until the agency chooses a pricing strategy.

## Phase-Two Expansion Options

After phase one works, possible expansions include:

- Add additional lead sources.
- Add CRM integration.
- Add scheduling link or booking workflow.
- Add approved auto-send follow-up.
- Add weekly owner summary.
- Add lead source reporting.
- Add proposal or estimate handoff automation.

## Current Recommendation

For the first qualified Dallas home-services opportunity, lead with Package 2 if the prospect confirms one primary estimate or service request source. Use Package 1 if they remain curious but vague, and Package 3 if they already have a CRM but admit follow-up still gets stale.
